La primer fase, segun el video, es el analisis. En el documento de ciclo de vida de una base de datos lo llama "Estudio inicial de una base de datos". Esta fase se debe comenzar cuestionando al usuario sobre el sistema. De esta forma se averiguaran que caracteristicas y restricciones extras demanda el usuario. También se armará una especificación funcional y un análisis estructurado con los requerimientos. El usuario debe luego revisar que todo este bien en estos documentos. Es posible que sea util desarrollar prototipos rapidos como parte del analisis de que requerimientos inesperados pueden terminar apareciendo.

La segunda fase es el "Diseño de una base de datos", en el que se debe definir definitivamente la estructura y forma definitiva de un sistema, o en nuestro caso de una base de datos. Se arman gráficos y diagramas con la estructura y se hace una especificación detallada con toda la documentacion del programa, sistema, o base de datos planificada. Se debe definir como se ingresaran y extraeran los datos, también conocido como el diseño externo de datos.

En la fase de desarrollo del video ocurren dos fases de las del documento: la denominada "Implementacion y carga" y la que lleva el nombre de "operacion" por el documento del classroom. La primera de estas dos consiste en varios pasos, uno siendo la adquisición del software. Esto se puede hacer a través de una compra, consiguiendo un software propietario, a través del codigo abierto, donde estos sistemas se pueden modificar como sea necesario, o por el desarrollo mismo del software, con los pasos del diseño, la codificación, y la depuracion. Para el caso de las base de datos, normalmente se usan gestores de base de datos ya hechos, ya sean abiertos o comprados, y es rado desarrollar uno nuevo. Por lo tanto, para el caso de esta materia, lo que importa principalmente es el diseño de las bases de datos mismas, sus estructuras, etc. Si es común tener que hacer sistemas de interacción con las bases de datos, se todas maneras.
Otra parte de la implementación es la adquisición del hardware. También se puede usar hardware existente, y se debe analizar si convine mejorar un sistema que ya está, como una red, o si se crea algo de 0. 
La fase de operación, segun el documento, también correspondería a el desarrollo, ya que en este se desarrolla y se suministra la documentación del programa. Este trabajo no le corresponde a los programadores.

La fase de prueba, que también la tenemos como "Prueba y evaluación", se usan datos de muestra para evaluar y verificar el funcionamiento y la seguridad del sistema o la base de datos. Los usuarios también lo probarán y harán observaciones sobre cosas que pueden faltar por mejorar antes de que se ponga en uso.

La última fase es la del mantenimiento y la evolución. En esta fase post-depliegue de la base de datos solo se debe mantener y seguir por si algún cambio termina siendo necesario o si algo se termina rompiendo.
